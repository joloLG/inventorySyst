Manual Configuration

You can manually configure the system by editing the configuration files:

Option 1: Edit config.js directly

// js/config.js const SUPABASE_URL = 'https://your-project-id.supabase.co'; const SUPABASE_ANON_KEY = 'your-anon-key-here';

Option 2: Use browser localStorage

// Open browser console and run: localStorage.setItem('inventory_config', JSON.stringify({ supabase: { url: 'https://your-project-id.supabase.co', anonKey: 'your-anon-key-here' } }));

Option 3: Set window.env before loading

// In your HTML before loading scripts: window.env = { SUPABASE_URL: 'https://your-project-id.supabase.co', SUPABASE_ANON_KEY: 'your-anon-key-here' };

Environment Variables Setup

For Development (Node.js)

Create a `.env` file in your project root:

# .env SUPABASE_URL=https://your-project-id.supabase.co SUPABASE_ANON_KEY=your-anon-key-here SUPABASE_SERVICE_ROLE_KEY=your-service-role-key-here APP_NAME=Inventory Management System NODE_ENV=development

For Production

Set environment variables in your hosting environment:

# Example for various platforms export SUPABASE_URL="https://your-project-id.supabase.co" export SUPABASE_ANON_KEY="your-anon-key-here" export SUPABASE_SERVICE_ROLE_KEY="your-service-role-key-here" export NODE_ENV="production"

For Static Hosting (Vercel, Netlify, etc.)

Most static hosts provide environment variable settings in their dashboard:

  • Vercel: Project Settings → Environment Variables
  • Netlify: Site settings → Build & deploy → Environment
  • GitHub Pages: Use build process or embed in script
